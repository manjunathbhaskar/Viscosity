Cognitive Dissonance — The Math

Pure Arithmetic. No LLM Guessing.

Dissonance =
0.3 × The Trap +
0.4 × The Blindspot +
0.3 × The Chaos
The Trap: How one-sided is the crowd? (sentiment extremity)
The Blindspot: Consensus strength vs. kill-shot feasibility gap
The Chaos: Sentiment variance — do reviewers agree?

End-to-End Data Flow

One founder, seven APIs firing in parallel, 29 agents reasoning, 155 reviewers debating, one decision.

1. POST /api/source

Input: founder_name, company_name, topic

Parallel: GitHub auto-discover → X API → arXiv search → PatentsView → Product Hunt → website scrape → Tavily (×3)

2. Cold-Start Score + 3-Axis

Process signals: shipping cadence, completion rate, response quality

Three independent axes: Founder (68 ±16) | Market (55 ±17) | Idea-vs-Market (72 ±21)

3. POST /api/screen

Swarm activation: 29 analyst agents (keyword-triggered) + 155 reviewer personas (mode-dependent)

Turbo (25) / Standard (50) / Deep (155) — each reviewer reacts in one gut-feeling sentence + sentiment score

4. Dissonance Arithmetic (Pure Math)

No LLM guessing. Pure calculation: 0.3×Trap + 0.4×Blindspot + 0.3×Chaos

Output: Dissonance score (0-1), Bull % (crowd consensus), Kill-shot risk

5. Synthesis + Memo Generation

LLM call (Tier 1/2/3): Linchpin discovery, pressure-point extraction, prediction range, confidence calibration

Output: Decision-ready memo (mandatory sections only), investment recommendation, risk dashboard

6. POST /api/deals

Decision-ready queue: founder + company + scores + scenarios + confidence intervals + traceability links

Ready for: VC partner review, follow-on diligence, thesis fit check, validation agent
